FAQ

Im Prinzip jeder! Auf GitHub wird der bisherige Code zur Verfügung gestellt, so dass ihn jeder runterladen und gerne auch erweitern und/oder verbessern kann!

Es gibt drei Optionen, zum einen kann eine Liste hochgeladen werden und eine auf Property Sets basierte mvdXML wird erstellt. Dann ist es möglich, eine mvdXML und ein IFC Modell hochzuladen und zu prüfen. Und schließlich kann eine Tabelle und ein IFC Modell hochgeladen werden und es findet direkt ein Checking des IFC Modells nach den in der Tabelle angegebenen Kriterien statt

Tabellenformate: .csv, .xlxs

XML Formate: .mvdxml

Modellformat: .ifc

Mit diesem Tool lassen sich Property Sets, Properties und entsprechende Values prüfen. Die Vorgabe entspricht der Tabellenstrucktur der Bergischen Universität Wuppertal und ist im Reiter "Über das Tool" Abgebildet.

Da gibt es verschiedene Möglichkeiten. In den bisherigen Umsetzungen braucht es noch Optimierungen für einen problemlosen Prozessablauf, außerdem müssen Implementierungen optimiert werden. Über die bisherigen Umsetzungen hinaus, besteht die Möglichkeit weitere Entwicklungen wie IDM mit einzubeziehen und gesamte Prozesse in einer idmXML abzubilden. Abhängig von den Entwicklungen des IDS Projektes besteht auch hier die Möglichkeit Ideen unterzubringen. Auch andere Grundlagen wie mvdXML können auf ihre aktuellen Funktionen hin überprüft und müssen im Zweifel erweitert werden.
